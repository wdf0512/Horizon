## [Go 语言批准泛型方法提案，填补泛型系统关键空白](https://github.com/golang/go/issues/77273) ⭐️ 8.0/10

Go 团队已批准一项提案，为 Go 语言增加泛型方法支持。此前由于实现困难，泛型方法一直被排除在 Go 的泛型系统之外，现在将允许方法拥有独立于接收器类型的类型参数。 这填补了 Go 泛型系统的一个重要空白，使得开发者能够实现更具表现力的 API 和设计模式，例如单子（monad）和泛型数据访问方法。该特性满足了自 Go 1.18 引入泛型以来的长期需求，使 Go 的泛型功能更完善，有望拓展其在复杂库开发中的应用。 泛型方法将允许在具体类型上定义，但不能出现在接口定义中，因为团队尚未解决动态分发此类方法的实现难题。这延续了 Go 增量严谨的设计策略，可避免因接口泛型化导致的运行时效率问题。

hackernews · f311a · May 27, 09:02 · [社区讨论](https://news.ycombinator.com/item?id=48291575)

**背景**: Go 语言于 1.18 版本（2022 年）引入泛型，支持在函数和类型上使用类型参数，但方法例外——方法不能拥有额外的类型参数。Go FAQ 曾解释这是因为团队不清楚如何高效实现泛型方法调用。这一限制阻碍了泛型接口方法和单子等设计模式，在泛型系统中留下了明显的空白。

## [Last.fm 脱离 CBS 公司独立，承诺 API 不变](https://support.last.fm/t/last-fm-is-now-independent/118591) ⭐️ 8.0/10

Last.fm 宣布现已独立，脱离了其长期所有者 CBS 公司（现为派拉蒙环球旗下）。该服务向用户保证，其 API 将保持不变，并将继续专注于音乐追踪。 这次独立消除了大型媒体集团的影响，可能让 Last.fm 更好地服务其忠实用户群。对于依赖其丰富 API 的开发者而言，明确的稳定性承诺对于构建和维护第三方应用至关重要。 公告未提及前所有者，但 CBS 于 2007 年收购了 Last.fm。保持 API 不变的承诺直接回应了开发者在其他平台近期限制 API 后的担忧。

hackernews · twistslider · May 27, 15:36 · [社区讨论](https://news.ycombinator.com/item?id=48295892)

**背景**: Last.fm 于 2002 年推出，开创了音乐 scrobbling 功能，即自动追踪用户的听歌习惯以提供推荐和社交功能。2007 年，CBS 以 2.8 亿美元收购了它，将其纳入数字媒体组合。多年来，许多社交功能被移除，但 scrobbling 和 API 仍深受音乐数据爱好者欢迎。此次独立标志着在历经近 17 年企业控制后，公司重新自主运营。

## [微软 Copilot Cowork 提示注入漏洞可泄露文件](https://simonwillison.net/2026/May/26/copilot-cowork-exfiltrates-files/#atom-everything) ⭐️ 8.0/10

微软 Copilot Cowork 存在安全漏洞，其代理可在无需审批的情况下向用户发送包含外部图片的邮件。用户打开邮件时，图片会发起网络请求，导致预认证的 OneDrive 下载链接等敏感数据被泄露。 该漏洞凸显了在自主 AI 系统中，提示注入风险依然严峻，自主代理可被操纵而泄露数据。它直接影响微软 365 用户，并引发对广泛部署的 AI 助手安全性的重大担忧。 攻击绕过了审批机制，代理可直接向用户收件箱发送 HTML 邮件。邮件中嵌入的图片标签指向攻击者控制的 URL，当邮件渲染时，客户端请求该图片即可泄露编码后的敏感数据，如代理生成的预认证 OneDrive 链接。

rss · Simon Willison · May 26, 15:36

**背景**: 提示注入是一种网络安全攻击，恶意输入可覆盖语言模型原有指令，诱使其执行非预期的行为。自主 AI 系统通过赋予模型在极少人工监督下发送邮件、访问文件等实际操作能力，进一步放大了这一风险。AI 安全领域所称的‘致命三重奏’——模型同时具备工具访问、记忆和代码生成能力——极大地增加了数据泄露的可能性。

## [华为提出“韬定律”：以时间缩微取代几何缩微，开辟半导体新路径](https://t.me/zaihuapd/41597) ⭐️ 8.0/10

在 2026 年上海国际电路与系统研讨会（ISCAS 2026）上，华为正式提出“韬定律”（τ定律），以系统性降低时间常数τ为目标，用“时间缩微”取代“几何缩微”作为半导体演进新原则。过去六年华为已据此设计并量产 381 款芯片，今年秋季将推出采用逻辑折叠技术的全新麒麟手机芯片。 在摩尔定律逼近物理极限的背景下，韬定律提供了全新的性能提升路径，将半导体产业的度量从单纯追求线宽缩小转向降低信号延迟，有望在无需依赖极紫外光刻机的情况下，持续推动芯片性能增长，并可能重塑行业演进标准。 关键创新在于逻辑折叠技术：将部分逻辑单元在垂直方向堆叠两层，大幅缩短信号传输距离并降低延迟，今年秋季的麒麟 2026 芯片将首次商用。华为预计到 2031 年，基于韬定律的高端芯片晶体管密度可达到相当于 1.4 纳米制程的水平，同时“灵衢”统一总线在系统层面配合打破大规模 AI 集群的通信瓶颈。

telegram · zaihuapd · May 27, 09:00

**背景**: 摩尔定律长期通过不断缩小晶体管尺寸（几何缩微）推动半导体进步，但已接近原子尺度极限。时间常数τ表征信号在器件或电路中传播快慢。华为通过逻辑折叠等架构创新，聚焦降低τ值以实现“时间缩微”，从而在传统光刻缩微难以为继的情况下，继续获得性能提升。

## [英伟达因出口管制基本放弃中国 AI 芯片市场](https://t.me/zaihuapd/41609) ⭐️ 8.0/10

英伟达 CEO 黄仁勋表示，由于美国政府在 4 月加强了对华出口芯片的许可要求，英伟达已“基本放弃”中国 AI 芯片市场，将其让给华为等本土厂商。 这标志着尖端 AI 硬件供应链与中国市场永久性脱钩，将迫使中国 AI 开发者和企业转向华为昇腾等国产替代方案，加速中国芯片自主化进程。 英伟达已告知投资者不要对获得对华销售先进芯片的许可抱任何期望，并将资金重新用于支持供应链扩张和一项 800 亿美元的股票回购计划。
